Permission-Klassen

read_only: nur Lesezugriff, immer erlaubt.
write_local: schreibt auf das Workbench-Host (Shell, Files).
write_remote: schreibt auf entfernte Hosts via SSH.
destructive: löscht Daten, killt Prozesse, force-cancels. Nur in Work-Sessions.

Die effektive Tool-Sicht pro Agent wird per Session-Policy gefiltert: chat = nur read_only, research = read_only + write_local, work = alles.
